# vol_1 翻译指令（每个翻译进程必读）

把指定的【一个】缅文源文件翻译为中文，单遍到位（直接逐行对齐 + 术语合规），写入指定目标目录。

## 输入
- 缅文源文件：底本，逐句逐行翻译它。
- 英文参考：见 manifest 该行 english_ref 列，仅用于辅助理解长难句，**禁止直译英文**。
- 术语子集：见 manifest 该行 subset 列（pali→中文）。
- 勘误表：`glossary/errata.tsv`。

## 规则
1. **底本=缅文**，译成现代汉语、清晰易懂的书面语；禁文言文/半文半白/古译经腔。
2. **逐行对齐（硬约束）**：中文文件行数必须与源文件完全相同；中文第 N 行 = 源第 N 行的中文译文；源空行↔中文空行，位置完全一致；**禁止合并/拆分/折行/增删空行**；一行很长也保持一行。文件末尾换行与源一致。
3. **术语**：查术语子集；术语表有的用表中用词。**例外**：`glossary/errata.tsv` 中的错误条目不采用，用正确译法。术语表没有的词，按知识库翻译并标注巴利（可作回填候选）。
4. **专有名词**（人名/地名）用术语表的巴利音译。
5. **术语首现**：术语在【本文件】首次出现时写「中文（pali）」，其后仅用中文。
6. **纯巴利句/行**：直接给罗马巴利转写，不翻译；缅文字体的巴利优先从英文参考取罗马转写。
7. **书名**固定为《去向涅槃之道》。
8. 保留 markdown 标记（#标题、列表、<r>、<br> 等）在对应行。

## 输出与自验
- 目标文件名 = 把缅文源文件名翻译为中文，**保留前面的 `[页码]` 前缀并原样不动**。
  注意 vol_1 的前缀字母在【括号内】且可能补零，如 `[186a]`、`[002a]`、`[016b]`、`[769]`——必须一字不差地保留该方括号前缀，**文件名内不加巴利**。写入指定目标目录。
- 自验直到通过：
  - `python3 tools/check_lines.py "<源>" "<目标>"` → 必须 ✓
  - `python3 tools/check_terms.py "<目标>"` → 仅允许残留「未知巴利/勘误例外/书名」类。
- 回报：目标路径、行数是否对齐、首现术语数、纯巴利块数、存疑处。
